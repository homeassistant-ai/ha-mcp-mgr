## Core Beliefs

### Tool Design
- Names should be self-documenting
- Parameters should have sensible defaults
- Errors should guide recovery

### User Experience
- Consistency reduces cognitive load
- Discoverability enables adoption
- Progressive disclosure manages complexity

### Documentation
- Examples are more valuable than descriptions
- Real-world scenarios resonate
- Troubleshooting anticipates user pain
